### 금주 프로젝트 수행 내용

---

-   김지용
    -   FE : 필수 프레임워크 수정, Header(로그인 시 알림창, 장바구니 배지 구현, Footer, 알림창 클릭 시 나오는 모달창 구현, 사용자 아이콘 클릭시 카탈로그 모달창 구현, 검색바, 검색 바 클릭 시 검색창 구현) Side Sticky(장바구니, 최근에 본 상품 구현)
    -   DB : ERD 구성에 맞는 테스트용 더미 데이터 제작
    -   문서 : 기획서 최종 수정, API 명세서 작성(Header 부분)
    -   디자인 : Figma 디자인 중 빠진 내용 추가 구현 (회원가입 방식, 회원가입 후 사용자 개인 피부 상태 수집창, 검색 클릭 시 나오는 검색 창)
    -   BE : 구현된 Header의 필요 API 구현
-   김지훈
    -   문서 : API 명세서 작성(장바구니, 관심상품)
    -   DB : Table 생성 (CART, DIBS)
    -   BE : 장바구니(Cart) API 기능 구현 중
-   모란희

    -   FE : 관리자 메인 화면 작업, 관리자 주문 목록 기능 (작업중)
    -   DB : DB 테이블 생성(11개)
    -   BE : 관리자 주문 목록 REST API
    -   문서 : ERD 수정, wbs 수정

-   문지용
    -   **Bootstrap, swiper를** 이용한 홈페이지 제작
    -   관리자용 상품 crud 기능구현
-   박민우
    -   API 명세서 작성(운영자 상품 목록 조회, 등록, 수정, 삭제, 사용자 상품 목록 조회, 상품 상세 조회, 사용자 리뷰 목록 조회, 작성, 수정, 삭제, 좋아요, 신고, 성분 조회, 성분 상세 조회)
    -   제품 상세 조회 BE 구성, 로컬 개발 환경 설정
-   윤예주
    -   회원 관련 API 구조 및 MyBatis 연동 정상 빌드 완료
    -   `/member/health` 연결 테스트 성공
    -   로컬 환경에서 회원가입·로그인 API 테스트 준비 완료

### 차주 프로젝트 수행 계획

---

-   김지용
    -   BE : 구현된 Header 기능 연결 API 구현
    -   FE : 마이페이지 제작, 관리자 대시보드 제작 구현
    -   기획 : 회원 API 작성
-   김지훈

    -   BE : 장바구니 기능 완성 및 결제 페이지 연동

-   모란희:

    -   FE : 관리자 주문 목록 페이지, 관리자 주문 상세 페이지, 관리자 배송 조회 페이지
    -   BE : 관리자 배송 조회 REST API

-   문지용: 관리자용 상품 crud api 연결 가능 하도록 완성, 상품 페이징 처리, 성분 crud

-   박민우: 리뷰 crud, 좋아요,신고 BE, 상품 내 성분 조회 BE 구성, 제품 상세 페이지 FE 구성

-   윤예주: 회원가입 / 로그인 API 프론트 연동 테스트
