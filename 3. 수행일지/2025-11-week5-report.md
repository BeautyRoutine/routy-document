## 1. 프로젝트 수행 계획 및 현황

### 금주 프로젝트 수행 내용

---

-   공통

    -   실제 상품 데이터 수집

-   김지용
    -   BE : 마이페이지 관련 API 구성, 검색창, 최근에 본 상품 관련 API 구현, 랭킹페이지 API 구현(판매량 기준으로만 우선 구현, 피부타입별로는 추후 구현 예정,@EnableScheduling 어노테이션 활용으로 서버 기준 매일 오전7시 업데이트 )
    -   FE : 마이페이지(최근에 본 상품 페이지), 랭킹페이지 구현(카테고리 별 랭킹, 피부타입별 랭킹 구현)
    -   DB : 랭킹 관련 DB 테이블 구현(랭킹 캐시 값 업데이트 테이블,)
    -   배포 : CI/CD 기본 구축 관련 공부 및 배포 준비
-   김지훈
    -   FE: 결제 페이지 최종 완성 및 연동
    -   BE: 결제 연동 및 기능 구현
    -   DB: 리뷰 데이터 수집, 상품 등록
-   모란희
    -   BE : 카테고리 목록 API 추가. Map으로 값을 받는 서비스들 전부 DTO로 변경(api응답값에 필드명 스타일 통일 안되는 이슈 발생).
    -   FE : 사용자 apiBaseUrl 추가. 관리자 전체 API 경로 수정. 상품 관리 컴포넌트 이동 및 이미지 경로 수정. 로딩 이미지 공통 컴포넌트 분리.
    -   문서 : 관리자 상품, 주문 API 문서 전체 수정.
    -   BE,FE,문서 : 관리자 주문상세 상품&택배 목록 완성. 관리자 상품 등록 기능에 상세정보 추가 및 성분파싱후 자동매핑 기능 추가. 관리자 택배 추가&수정&삭제 기능 추가
-   문지용
    -   BE : 홈 화면에서 추천할 상품 추천기준 세분화 및 기능구현 BE (비로그인, 로그인, 피부타입 지정에 따라)
    -   FE : 홈 화면에서 추천할 상품 추천기준 세분화 및 기능구현 FE, 카테고리 - 전체 상품 페이지 연동
    -   문서 : 계획 세분화로 인한 WBS 수정
-   박민우
    -   BE : 상품 성분 전체 조회 기능, 프론트에서 요구하는 api 에 맞게 백엔드 수정
    -   FE: 제품 상세조회 페이지에서 리뷰, 개별 리뷰, 성분, 이미지, 제품 정보 구현
    -   DB,문서: 상품 DB 등록, API 문서 최신화
-   윤예주
    -   BE :카카오 연결 중, 피부스킨타입 설정
    -   FE : 로그인 , 회원가입 , 피부스킨타입 페이지 수정
    -   문서 :API 문서 전체 수정, 발표 자료 준비
